WHAT WAS BUILT
--------------

A full gateway connection layer that lets the Electron Fleet Manager connect
to multiple OpenClaw Gateway instances via WebSocket for real-time monitoring.

Architecture:
  Electron Main Process owns WS connections → exposes data via oRPC IPC →
  Renderer consumes via TanStack Query with real-time event streaming.

KEY ARCHITECTURE DECISIONS
--------------------------

1. WebSocket in main process only
   - ws library (not browser WebSocket) runs in Electron main process
   - All WS connections owned by GatewayManager singleton
   - Renderer never touches WebSocket directly

2. oRPC for all IPC
   - All data flows through typed oRPC procedures over MessagePort
   - Event streaming uses @orpc/shared EventPublisher → async generator → experimental_streamedOptions
   - Renderer invalidates TanStack Query caches when events arrive

3. Ed25519 device identity
   - Node crypto.generateKeyPairSync('ed25519') — no extra dependencies
   - Challenge-response signing per TRD Appendix B
   - Persisted via electron-store

4. Reconnection strategy
   - 800ms initial backoff, 1.7x multiplier, 15s max
   - Resets on successful hello-ok
   - Stops on auth-failed (user must fix token)
   - Sequence gap detection triggers state refresh

5. No custom hooks
   - All queries/mutations used directly in components per CLAUDE.md
   - Pattern: useQuery(orpc.gateway.list.queryOptions())

oRPC PROCEDURES
---------------
  gateway.list              — List all gateways with runtime state
  gateway.get               — Get single gateway by ID
  gateway.add               — Add new gateway (url, token, label)
  gateway.remove            — Remove gateway
  gateway.update            — Update gateway config
  gateway.testConnection    — Test connection before saving
  gateway.reconnect         — Force reconnect a gateway
  gateway.sessions          — List sessions for a gateway
  gateway.sessionUsage      — Usage stats for a session
  gateway.sessionUsageLogs  — Per-message usage logs
  gateway.chatHistory       — Chat messages for a session
  gateway.agents            — List agents on a gateway
  gateway.presence          — Connected devices/clients
  gateway.cost              — Cost summary for a gateway
  fleet.overview            — Fleet-wide stats (connected/disconnected counts, total sessions)
  events.subscribe          — Real-time event stream (async generator)
