# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in OpenClaw Swarm, please report it responsibly.

**Do not open a public issue.** Instead, use one of the following:

- **GitHub private vulnerability reporting**: [Report a vulnerability](https://github.com/kddige/openclaw-swarm/security/advisories/new)

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fix (if you have one)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix or mitigation**: Depends on severity, but we aim to address critical issues as quickly as possible

## Scope

This policy applies to the OpenClaw Swarm desktop application and its associated codebase. For vulnerabilities in OpenClaw Gateway, please report them to the [OpenClaw Gateway repository](https://github.com/openclaw/openclaw).

## Supported Versions

Only the latest release is supported with security updates.

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |
| Older   | No        |
